# RPG風マップ探索システム（第1段階: マップの土台）

学校の授業で制作するRPG風テキストアドベンチャーゲームの、最初の土台となる
「マップ探索システム」のみを実装したものです。会話・イベント・アイテム・
セーブなどはまだ実装していません。

## 動かし方

ES Modules (`<script type="module">`) を使っているため、`index.html` を
直接ダブルクリックして開くとブラウザのセキュリティ制限で読み込めません。
簡易サーバー経由で開いてください。

```bash
# Python がある場合
python3 -m http.server 8000

# Node がある場合
npx serve .
```

その後ブラウザで `http://localhost:8000` を開きます。
（3D描画にWebGLを使用しているため、WebGLに対応した一般的なブラウザ
[Chrome/Edge/Firefox/Safari等の最新版] が必要です。）

## 操作方法

| 操作 | キー |
|---|---|
| 移動 | `W A S D` または 矢印キー |
| デバッグ表示切替 | `F1` または画面右上のボタン |
| 全画面表示切替 | 画面右上の「全画面」ボタン |

ゲーム画面はブラウザの表示領域いっぱいに表示され、ウィンドウサイズを
変えると自動的に追従します。「全画面」ボタンを押すと、ブラウザの
Fullscreen APIを使ってOSレベルの全画面表示（ブラウザのUIも隠れる）に
切り替わります。

デバッグ表示をONにすると、マップ端の出入り口が地面に半透明の白い板として
表示されます。NPCは常に表示されており、名前ラベルが頭上に浮かんでいます。

## ディレクトリ構成と設計方針

```
index.html
css/style.css
js/
  constants.js        … タイルサイズ・移動速度・3Dカメラ設定などの定数
  main.js             … エントリーポイント
  vendor/
    three.module.min.js … Three.js本体（同梱・CDN不使用、MITライセンス）
  core/
    Game.js           … ゲームループ本体（今後: 状態管理を追加予定）
    Input.js          … キーボード入力
    Camera.js         … 3D追従カメラ
    Renderer.js       … 3D描画（見た目の変更はほぼここで完結）
  entities/
    Player.js         … プレイヤーの移動・当たり判定解決（2Dのタイル座標のまま）
  map/
    MapManager.js     … 現在のマップ保持・マップ切り替え
    Collision.js       … 当たり判定の純粋関数
  data/                … ロジックを一切含まない「データ」だけの層
    tileTypes.js       … 地形タイルの定義（色・通行可否）
    buildingTypes.js   … 建物の定義（今回は表示のみ、内部マップは未実装）
    objectTypes.js     … 看板・ベンチ等の定義（onInteractは今回null）
    npcTypes.js        … NPCの見た目バリエーション定義
    mapHelpers.js       … マップデータ作成の共通処理
    maps/
      index.js         … 全マップの登録簿（★新マップはここに追加）
      home.js, residential.js, shopping.js, school.js,
      park.js, library.js, shrine.js, forest.js, plaza.js
```

**ロジックとデータを分離**しているのがポイントです。
`js/core` `js/map` `js/entities` は「動かし方」のコードで、
`js/data` は「何がどこにあるか」のデータです。今後ストーリーや
NPCを追加するときは基本的に `js/data` 以下を触るだけで済みます。

### 3D描画の仕組み

マップは見た目こそ3Dになりましたが、**当たり判定・マップ切り替え・
プレイヤーの座標管理はすべて元の2Dタイル座標のまま**です
（`js/map` `js/entities` は3D化にあたって一切変更していません）。
3D化で変更したのは `js/core/Renderer.js` と `js/core/Camera.js` だけです。

- 地面: タイルの色をcanvasに描いてテクスチャ化し、1枚の平面に貼り付けたもの
- 木・岩・山・崖・橋: タイルの上に立てた3Dオブジェクト（`InstancedMesh`でまとめて描画し、同じ種類が何十個あっても軽く動くようにしている）
- 建物・設置物・NPC: 箱や円錐などの単純な形状を組み合わせた3Dオブジェクト
- カメラ: プレイヤーの向きには回転せず、常に同じ角度で後方・上空から追従する固定アングルの3人称カメラ

### 3Dのボリューム感を調整したい場合

`js/constants.js` の `HEIGHTS`（建物・木・NPCなどの高さ）や
`CAMERA_HEIGHT` / `CAMERA_DISTANCE`（カメラの高さ・距離）を
数値で調整するだけで、見た目の印象をまとめて変えられる。

## 現在実装済みの機能

- 3DのRPG風マップ、WASD/矢印キーでの滑らかな移動
- プレイヤーを後方・上空から追従する固定アングルの3Dカメラ
- タイルベースの当たり判定（建物・木・岩・川・崖などは通行不可、判定ロジック自体は2Dのまま）
- 複数エリア（主人公の家・住宅街・商店街・学校・公園・図書館・神社・森・広場）
- エリア間のマップ切り替え（マップ端の出口に触れると隣接マップへ移動）
- 建物・設置物（看板/ベンチ/街灯/ポスト/自動販売機/宝箱/花壇）を3Dオブジェクトとして配置
- NPC（合計18体）を各エリアに3Dキャラクターとして配置（見た目のみ・通行不可・会話は未実装）

## 今後の拡張方法

### 新しいマップを追加する
1. `js/data/maps/新マップ.js` を作成し、既存ファイル（例: `park.js`）を
   参考に `tiles` `buildings` `objects` `npcSpawns` `exits` を定義する
2. `js/data/maps/index.js` に import と登録を1行ずつ追加する
3. 既存マップの `exits` に、新マップへの出口を追加する

### 新しい建物・オブジェクトの種類を追加する
`js/data/buildingTypes.js` / `js/data/objectTypes.js` に定義を1つ追加し、
`js/core/Renderer.js` の `buildBuildingMesh` / `buildObjectMesh` に
対応する3D形状（箱・円柱・円錐などの組み合わせ）を追加する。

### NPCを追加する
各マップの `npcs` 配列に `placeNpc(...)` を1件追加するだけでよい。
見た目のバリエーションを増やしたい場合は `js/data/npcTypes.js` に
新しい種類を追加する。会話をさせたい場合は、各NPCが持つ
`onInteract`（今回はすべて null）を将来の会話システムから
呼び出すようにする想定。

### 会話・イベント・マルチエンディングを追加する（将来実装時）
`js/data/objectTypes.js` の各定義にある `onInteract: null` を、
イベント処理関数に差し替えることでフックできるように設計してある。
`js/core/Game.js` に `state`（`exploring` / `dialogue` / `menu` など）を
持たせ、会話中は移動処理を止める、という形で拡張する想定。

# RPG風マップ探索システム（第1段階: マップの土台）

学校の授業で制作するRPG風テキストアドベンチャーゲームの、最初の土台となる
「マップ探索システム」のみを実装したものです。会話・イベント・アイテム・
セーブなどはまだ実装していません。

## 動かし方

ES Modules (`<script type="module">`) を使っているため、`index.html` を
直接ダブルクリックして開くとブラウザのセキュリティ制限で読み込めません。
簡易サーバー経由で開いてください。

```bash
# Python がある場合
python3 -m http.server 8000

# Node がある場合
npx serve .
```

その後ブラウザで `http://localhost:8000` を開きます。

## 操作方法

| 操作 | キー |
|---|---|
| 移動 | `W A S D` または 矢印キー |
| デバッグ表示切替 | `F1` または画面下のボタン |

デバッグ表示をONにすると、マップ端の出入り口(白枠)と、将来NPCを配置する
予定地(ピンクの破線円)が表示されます。

## ディレクトリ構成と設計方針

```
index.html
css/style.css
js/
  constants.js        … タイルサイズ・移動速度などの定数
  main.js             … エントリーポイント
  core/
    Game.js           … ゲームループ本体（今後: 状態管理を追加予定）
    Input.js          … キーボード入力
    Camera.js         … 追従カメラ
    Renderer.js       … 描画（見た目の変更はほぼここで完結）
  entities/
    Player.js         … プレイヤーの移動・当たり判定解決
  map/
    MapManager.js     … 現在のマップ保持・マップ切り替え
    Collision.js       … 当たり判定の純粋関数
  data/                … ロジックを一切含まない「データ」だけの層
    tileTypes.js       … 地形タイルの定義（色・通行可否）
    buildingTypes.js   … 建物の定義（今回は表示のみ、内部マップは未実装）
    objectTypes.js     … 看板・ベンチ等の定義（onInteractは今回null）
    mapHelpers.js       … マップデータ作成の共通処理
    maps/
      index.js         … 全マップの登録簿（★新マップはここに追加）
      home.js, residential.js, shopping.js, school.js,
      park.js, library.js, shrine.js, forest.js, plaza.js
```

**ロジックとデータを分離**しているのがポイントです。
`js/core` `js/map` `js/entities` は「動かし方」のコードで、
`js/data` は「何がどこにあるか」のデータです。今後ストーリーや
NPCを追加するときは基本的に `js/data` 以下を触るだけで済みます。

## 現在実装済みの機能

- 2DトップビューのRPG風マップ、WASD/矢印キーでの滑らかな移動
- プレイヤー追従カメラ（マップ端でクランプ）
- タイルベースの当たり判定（建物・木・岩・川・崖などは通行不可）
- 複数エリア（主人公の家・住宅街・商店街・学校・公園・図書館・神社・森・広場）
- エリア間のマップ切り替え（マップ端の出口に触れると隣接マップへ移動）
- 建物・設置物（看板/ベンチ/街灯/ポスト/自動販売機/宝箱/花壇）の配置
- NPC（合計18体）を各エリアに配置（見た目のみ・通行不可・会話は未実装）

## マップの接続関係

```
              図書館
                |
商店街 ─ 住宅街   |
  |         |    |
  └── 広場 ──┼── 学校 ── 森
            |
           神社
            |
           公園
```

主人公の家 → 住宅街 → 商店街 → 広場、広場を中心に図書館・神社・学校へ、
さらに神社から公園、学校から森へ繋がっています。

## 今後の拡張方法

### 新しいマップを追加する
1. `js/data/maps/新マップ.js` を作成し、既存ファイル（例: `park.js`）を
   参考に `tiles` `buildings` `objects` `npcSpawns` `exits` を定義する
2. `js/data/maps/index.js` に import と登録を1行ずつ追加する
3. 既存マップの `exits` に、新マップへの出口を追加する

### 新しい建物・オブジェクトの種類を追加する
`js/data/buildingTypes.js` / `js/data/objectTypes.js` に定義を1つ追加し、
必要であれば `js/core/Renderer.js` の該当箇所に見た目を追加する。

### NPCを追加する
各マップの `npcs` 配列に `placeNpc(...)` を1件追加するだけでよい。
見た目のバリエーションを増やしたい場合は `js/data/npcTypes.js` に
新しい種類を追加する。会話をさせたい場合は、各NPCが持つ
`onInteract`（今回はすべて null）を将来の会話システムから
呼び出すようにする想定。

### 会話・イベント・マルチエンディングを追加する（将来実装時）
`js/data/objectTypes.js` の各定義にある `onInteract: null` を、
イベント処理関数に差し替えることでフックできるように設計してある。
`js/core/Game.js` に `state`（`exploring` / `dialogue` / `menu` など）を
持たせ、会話中は移動処理を止める、という形で拡張する想定。
